# Project 2 - AI Receptionist Generation Platform

## Positioning
**24/7 AI Front Desk for Dental Clinics**

## Executive Summary
Project 2 is a multi-tenant AI agent configuration platform for dental clinics.  
One **Master AI Core Brain** powers reasoning, conversation control, and orchestration across all tenants.  
Each clinic receives a **dynamically configured AI instance** generated from:
- Structured onboarding forms
- Guided AI configuration chat
- Integration settings and operational policies

Clinic knowledge is captured as structured, validated configuration data.

## Updated Architecture Direction

### Current Model
- Configuration-first architecture with deterministic schema outputs.
- Guided chat performs targeted extraction into canonical fields.
- Configuration completeness and validation gates control deployment.
- Runtime AI behavior is governed by strict structured context injection.

## Product Definition
The platform enables clinics to launch an AI receptionist that can:
- Answer calls 24/7 with clinic-specific tone and policies
- Book, reschedule, and cancel appointments using configured rules
- Resolve common questions from structured FAQ and policy libraries
- Escalate to human staff based on deterministic safety and business rules
- Produce auditable call outcomes and analytics

## Master AI + Configurable Tenant Model

| Component | Responsibility | Shared vs Tenant-Specific |
| --- | --- | --- |
| Master AI Core Brain | Reasoning, planning, policy enforcement, tool orchestration | Shared |
| Tenant Configuration Layer | Clinic profile, services, booking rules, tone, policies, FAQs, integrations | Tenant-specific |
| Runtime Session Context | Live call state, caller intent, temporary memory, action history | Session-specific |

Key principle: **the same core AI engine serves every clinic, while tenant behavior is controlled by structured configuration data.**

## System Philosophy

### 1) Configuration Over Content Dumps
- Prefer explicit fields, rule objects, and constrained values.
- Minimize ambiguity by enforcing typed schemas and validations.

### 2) Guardrailed Adaptability
- AI can be flexible in dialogue, but not in policy boundaries.
- Escalation and booking behavior are rule-based, not free-form.

### 3) Tenant Isolation by Design
- Every API call, query, event, and cache entry is tenant-scoped.
- Runtime context never leaks across tenants.

### 4) Continuous Improvement Without Fragmentation
- Core AI improvements are rolled out centrally.
- Tenant behavior remains stable through versioned configuration contracts.
